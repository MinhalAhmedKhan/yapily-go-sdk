Allowed values are : <br/><br/>- INITIATED - Payment process initiated<br/>- DECLINED - Payment process failed and will not proceed further<br/>- INSTITUTION_SUBMITTED - Payment institution submitted<br/>- EMBEDDED_CREDENTIAL_REQUESTED - For embedded banks, a UI element to collect user credentials was displayed<br/>- CREDENTIALS_ERROR - embedded credentials refused by institution<br/>- AUTHORISATION_INITIATED - All details required for payment initiation have been collected<br/>- VALIDATION_COMPLETED - The payment payload was validated successfully<br/>- VALIDATION_FAILED - The payment data provided failed validation<br/>- AUTHORISATION_CREATED - Payment authorisation request created with Institution<br/>- EMBEDDED_CODE_REQUESTED - For embedded banks, a UI element to collect SCA for initiated consent was displayed<br/>- EMBEDDED_TYPE_REQUESTED - For embedded banks, a UI element to allow the user to select their preferred SCA method for this consent authorisation was displayed<br/>- DECOUPLED_AUTHORISATION - For embedded banks, decoupled authorisation was initiated by the bank<br/>- EMBEDDED_CODE_COLLECTED - For embedded banks, SCA code was collected for consent authorisation<br/>- EMBEDDED_TYPE_SELECTED - For embedded banks, preferred SCA method was selected for consent authorisation<br/>- PRE_AUTHORISED - pre authorisation was initiated by bank<br/>- CONSENT_POLLING_STARTED - We start polling the bank for consent authorisation status<br/>- CONSENT_POLLING_ENDED - We finish polling the bank for consent authorisation status<br/>- AUTHORISED - Payment authorisation completed<br/>- AUTHORISATION_FAILED - Payment authorisation failed and will not proceed further<br/>- AUTHORISATION_REJECTED - Payment authorisation rejected and will not proceed further<br/>- SUBMITTED - Payment execution created and submitted to Institution<br/>- SUBMITTED_AUTO - Payment execution created and submitted to Institution<br/>- ACCEPTED - Payment execution accepted by Institution and awaiting settlement<br/>- REJECTED - Payment or Authorisation request rejected by Institution and will not proceed further<br/>- SETTLEMENT_COMPLETED - Payment settlement completed<br/>- STATUS_POLLING_STARTED - Payment status polling started<br/>- STATUS_POLLING_ENDED - Payment status polling ended<br/>- MERCHANT_ACKNOWLEDGED - Payment acknowledgement received from merchant<br/>- FINISHED - Payment process completed |
